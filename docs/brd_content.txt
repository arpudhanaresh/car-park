1. Cancel Booking Functionality
The system will allow users to cancel a confirmed parking lot booking subject to configurable business rules. This feature enhances user flexibility while protecting the property management from last-minute cancellations.
1.1 Business Rules & Validations
Users can cancel bookings only before the booking start date/time.
Cancellation requests within the last X hours (configurable, e.g., 2 hours before start) will not be allowed.
For refundable bookings, system will trigger refund via RinggitPay API subject to refund policy.
Partial refund percentage can be configured based on cancellation window (e.g., 100% if >24h, 50% if <24h).
Once cancelled, lot becomes immediately available for rebooking.
System logs cancellation event with timestamp and user ID for audit.
Email notification (cancellation confirmation) sent to user and admin.
Non-refundable bookings will show alert message before confirmation.
